# Surfaces

Surfaces create grouping and depth. Ponti favors quiet structure over decorative elevation.

## Levels

1. **Canvas** — the page background.
2. **Panel** — a contained region, list, or quiet control area.
3. **Raised** — an overlay or surface that temporarily sits above the page.
4. **Inset** — a recessed region such as an input or code well.

Use borders when they clarify ownership or separation. Use blur when it preserves context behind an attention-demanding surface. Ponti’s web surfaces do not use decorative shadows by default; an exception requires a documented depth problem.

## Material guidance

Glass is a surface treatment, not a component category. It requires a readable fallback, a visible edge, and sufficient contrast against its backdrop. It must not make content harder to read or cause an interaction to disappear.

Radii express continuity: small for controls, medium for standard containers, large for overlays, and full for pills or circular controls. Components use the shared radius tokens rather than inventing local values.
