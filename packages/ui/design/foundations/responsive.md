# Responsive Behavior

Design mobile-first. A desktop layout is an expansion of a usable narrow layout, not a separate composition.

## Rules

- Content must remain readable at 390px without horizontal overflow.
- Interactive targets remain at least 44px in both dimensions.
- Metadata wraps before it compresses primary content.
- Rows can stack; controls can move into a sheet or flow region.
- Horizontal scrolling is reserved for content that benefits from a strip, such as screenshots or media.
- Fixed viewport-height layouts must account for mobile browser chrome and dynamic viewport units.
- Desktop overlays may become bottom sheets or stacked panels on mobile.

Use responsive changes to preserve hierarchy, not to add more content.
