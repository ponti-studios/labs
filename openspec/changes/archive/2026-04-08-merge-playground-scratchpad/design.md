## Context

The `apps/playground/app/` directory contains a nested `scratchpad/` subdirectory that is itself a complete React Router app with separate routing, components, and styling. This structure was likely created to experiment with isolated contexts, but now serves as an obstacle. The playground should be a single unified app where all experiments coexist.

**Current Structure:**
```
apps/playground/app/
├── app/                    # Main React Router app
│   ├── root.tsx, routes.ts, env.ts, app.css
│   ├── components/         # ~24 components
│   ├── routes/            # ~30 files
│   └── ...
└── scratchpad/             # Nested React Router app
    ├── root.tsx, routes.ts, env.ts, app.css
    ├── components/         # ~20 components (partially overlapping names)
    ├── routes/            # ~40 files (partially overlapping)
    ├── rust/              # Rust experiments
    └── ...
```

## Goals / Non-Goals

**Goals:**
- Flatten the scratchpad into the main app directory
- Eliminate duplicate/conflicting files
- Move rust/ to repo root where it belongs
- Preserve all experimental functionality

**Non-Goals:**
- Rewriting or refactoring existing components (just relocating)
- Changing the design system (keep main app's design tokens)
- Modifying any application logic

## Decisions

### 1. Conflicting Files Strategy

| File | Decision | Rationale |
|------|----------|-----------|
| `app.css` | Keep main app version | Main app has design token system, scratchpad has simpler styles |
| `root.tsx` | Keep main app version | Includes `PrefetchProvider` which scratchpad lacks |
| `routes.ts` | **Merge** both files | All routes needed; concatenate with deduplication |
| `env.ts` | **Merge** schemas | Both define env vars; `OPENAI_API_KEY` from scratchpad maps to `VITE_OPENAI_API_KEY` |

### 2. Duplicate Route Files (identical or near-identical)

These files exist in both directories with minor differences. Keep main app version, delete scratchpad version:

- `home.tsx` - main uses `@pontistudios/ui`, scratchpad uses local terminal
- `tfl.tsx` - functionally identical
- `svg-glass-test.tsx` - identical
- `api.tfl.ts` - identical

### 3. Route Files with Different Content

- `border-linear-gradient.tsx` - main uses `@pontistudios/ui`, scratchpad uses `~/components/ui`. **Merge** by using scratchpad's component imports (they're the same components, just different import paths)
- `infinite-header/` - completely different implementations. **Keep both** in merged routes with distinct file arrangements

### 4. Component Name Collisions

Same-name components (9): ApiKeyTester, CoronaLayout, eye-follow, MarkdownUpload, MorphingMenuIcon, Navigation, PostItem, QueryProvider, Sidebar, TFLMap, UploadDocumentsForm, VectorSearch, count-up, country-picker, covid, gradient-controls, terminal, ui.

**Strategy**: Keep main app versions. Scratchpad's versions are either identical or experiments that don't need to be preserved as separate variants.

### 5. Scratchpad-Unique Content to Move

| Path | Destination |
|------|-------------|
| `scratchpad/routes/corona*.tsx` | `app/routes/` |

| `scratchpad/lib/covid*` | `app/lib/` |
| `scratchpad/db/` | `app/db/` |

### 6. Rust Directory Move

Move `apps/playground/app/scratchpad/rust/` to `/Users/charlesponti/Developer/labs/rust/`.

**Note**: The scratchpad's `+types/` directory is auto-generated by React Router and should not be moved manually.

## Risks / Trade-offs

| Risk | Mitigation |
|------|------------|
| Route conflicts after merge | Review merged routes.ts thoroughly; some scratchpad routes may need renaming |
| Lost component variants | Main app versions are more mature; scratchpad experiments preserved in spirit |
| CSS conflicts | Main app's design system takes precedence; scratchpad styles are simple enough to not conflict |
| rust/ import references break | Update any scripts referencing old path |
