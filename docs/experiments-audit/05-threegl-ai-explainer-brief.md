# ThreeGL AI Explainer — Product Brief

## Purpose

Make an invisible computational process spatial, dynamic, and emotionally legible.

## Core model

Particles move through a sequence of layers. Each layer represents a stage of transformation, not merely a wall in a 3D scene. The visual system communicates progression, change, and accumulation.

## Core loop

Watch the flow, adjust the complexity or speed, and observe how the same transformation model behaves under different conditions.

## Novelty

It makes a process usually explained through static architecture diagrams feel like a living system without requiring the visitor to inspect the implementation.

## Discovery and proof

The visitor should understand: “Intelligence is not magic in a box. It is complexity emerging through repeated transformation.” The concept works when they can explain what the layers and particles represent after a short encounter.

## Non-goals

This is not a generic particle visualizer or a complete technical inventory of transformer architecture. Scientific precision matters, but explanatory clarity matters more than adding visual machinery.
